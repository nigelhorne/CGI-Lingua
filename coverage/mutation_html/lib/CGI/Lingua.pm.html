   54: 
   55: No longer does your website need to be in English only.
   56: CGI::Lingua provides a simple basis to determine which language to display a website.
   57: The website tells CGI::Lingua which languages it supports.

 1945: =head1 LIMITATIONS
 1946: 
 1947: =over 4
 1948: 
 1949: =item * B<is_rtl() covers primary-script RTL languages only>
 1950: 
 1951: C<is_rtl()> returns true for the 10 ISO 639-1 codes whose overwhelmingly
 1952: dominant script is right-to-left.  Languages with script variants (e.g.
 1953: Azerbaijani C<az>, which uses Latin in modern Azerbaijan but Arabic in Iran)
 1954: are treated as LTR.  If you serve content in multiple scripts of the same
 1955: language, inspect the sublanguage or Accept-Language header directly.
 1956: 
 1957: =item * B<plural_category() uses embedded CLDR rules, not Locale::CLDR>
 1958: 
 1959: The embedded rules cover ~70 languages and truncate fractional C<$n> to an
 1960: integer.  For full CLDR v42 accuracy (including fractional forms and
 1961: languages not in the table) install and use C<Locale::CLDR> directly.
 1962: 
 1963: =item * B<Logger must be a blessed object>
 1964: 
 1965: The C<logger> parameter is documented as accepting a code ref, array ref, or
 1966: filename, but the current implementation calls C<< $logger->$level() >> and will
 1967: die on non-blessed values.  Wrap alternative logger types in a
 1968: C<Log::Abstraction> instance before passing them to C<new()>.
 1969: 
 1970: =item * B<es-419 sublanguage returns undef>
 1971: 
 1972: Three-part regional codes such as C<es-419> (Latin American Spanish) do not
 1973: resolve to a C<sublanguage()> value because ISO 3166-1 does not define '419'.
 1974: This is a known limitation of the Locale::Object layer.
 1975: 
 1976: =item * B<Whois lookups are slow and unreliable>
 1977: 
 1978: Without C<IP::Country>, C<Geo::IP>, or C<Geo::IPfree> installed, C<country()>
 1979: falls back to Whois queries against live RIPE/ARIN/IANA servers.  These can
 1980: time out under load.  Install at least one local geo-database module and enable
 1981: the CHI cache to avoid this.
 1982: 
 1983: =item * B<Sub::Private not yet enforced>
 1984: 
 1985: The C<_*> private methods are currently accessible from outside the package.
 1986: C<Sub::Private> should be added to enforce encapsulation once white-box tests
 1987: are updated to call only the public API.
 1988: 
 1989: =item * B<IPv4-mapped IPv6 addresses are normalised to IPv4>
 1990: 
 1991: C<REMOTE_ADDR> values in the form C<::ffff:a.b.c.d> (RFC 4291 section 2.5.5)
 1992: are silently rewritten to the embedded C<a.b.c.d> IPv4 address before any
 1993: geo-lookup.  This is correct for country detection purposes but means the raw
 1994: address string is not preserved in cache keys or log messages.
 1995: 
 1996: =item * B<EU country code is irresolvable (with one exception)>
 1997: 
 1998: IP addresses that Whois reports as country C<EU> are mapped to C<'Unknown'>
 1999: unless they fall within Baidu's known subnet (RT-86809).  There is no ISO
 2000: 3166-1 country code for the European Union.
