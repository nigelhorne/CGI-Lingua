   53: offering extensive language/country detection strategies.
   54: 
   55: No longer does your website need to be in English only.
   56: CGI::Lingua provides a simple basis to determine which language to display a website.
   57: The website tells CGI::Lingua which languages it supports.
   58: Based on that list CGI::Lingua tells the application which language the user would like to use.
   59: 
   60:     use CGI::Lingua;

 1666: =head1 LIMITATIONS
 1667: 
 1668: =over 4
 1669: 
 1670: =item * B<Accept-Language left-to-right scan ignores q-values>
 1671: 
 1672: The second and third pass in C<_accept_language_match()> scan the header
 1673: left-to-right and ignore quality (C<q=0.x>) values.  A header such as
 1674: C<de;q=0.9,en;q=0.1> on a site that only supports C<en> would currently
 1675: fail to fall back to English.  Use C<I18N::AcceptLanguage> passes only when
 1676: possible.
 1677: 
 1678: =item * B<Logger must be a blessed object>
 1679: 
 1680: The C<logger> parameter is documented as accepting a code ref, array ref, or
 1681: filename, but the current implementation calls C<< $logger->$level() >> and will
 1682: die on non-blessed values.  Wrap alternative logger types in a
 1683: C<Log::Abstraction> instance before passing them to C<new()>.
 1684: 
 1685: =item * B<es-419 sublanguage returns undef>
 1686: 
 1687: Three-part regional codes such as C<es-419> (Latin American Spanish) do not
 1688: resolve to a C<sublanguage()> value because ISO 3166-1 does not define '419'.
 1689: This is a known limitation of the Locale::Object layer.
 1690: 
 1691: =item * B<Whois lookups are slow and unreliable>
 1692: 
 1693: Without C<IP::Country>, C<Geo::IP>, or C<Geo::IPfree> installed, C<country()>
 1694: falls back to Whois queries against live RIPE/ARIN/IANA servers.  These can
 1695: time out under load.  Install at least one local geo-database module and enable
 1696: the CHI cache to avoid this.
 1697: 
 1698: =item * B<Sub::Private not yet enforced>
 1699: 
 1700: The C<_*> private methods are currently accessible from outside the package.
 1701: C<Sub::Private> should be added to enforce encapsulation once white-box tests
 1702: are updated to call only the public API.
 1703: 
 1704: =item * B<IPv4-mapped IPv6 addresses are normalised to IPv4>
 1705: 
 1706: C<REMOTE_ADDR> values in the form C<::ffff:a.b.c.d> (RFC 4291 section 2.5.5)
 1707: are silently rewritten to the embedded C<a.b.c.d> IPv4 address before any
 1708: geo-lookup.  This is correct for country detection purposes but means the raw
 1709: address string is not preserved in cache keys or log messages.
 1710: 
 1711: =item * B<EU country code is irresolvable (with one exception)>
 1712: 
 1713: IP addresses that Whois reports as country C<EU> are mapped to C<'Unknown'>
 1714: unless they fall within Baidu's known subnet (RT-86809).  There is no ISO
 1715: 3166-1 country code for the European Union.
 1716: 
 1717: =back
