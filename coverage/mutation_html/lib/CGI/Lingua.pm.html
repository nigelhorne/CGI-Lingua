   52: CGI::Lingua is a powerful module for multilingual web applications
   53: offering extensive language/country detection strategies.
   54: 
   55: No longer does your website need to be in English only.
   56: CGI::Lingua provides a simple basis to determine which language to display a website.
   57: The website tells CGI::Lingua which languages it supports.
   58: Based on that list CGI::Lingua tells the application which language the user would like to use.
   59: 
   60:     use CGI::Lingua;

   62:     my $l = CGI::Lingua->new(['en', 'fr', 'en-gb', 'en-us']);
   63:     my $language = $l->language();

   65: 	print '<P>Hello</P>';

 1644: =head1 LIMITATIONS
 1645: 
 1646: =over 4
 1647: 
 1648: =item * B<Accept-Language left-to-right scan ignores q-values>
 1649: 
 1650: The second and third pass in C<_accept_language_match()> scan the header
 1651: left-to-right and ignore quality (C<q=0.x>) values.  A header such as
 1652: C<de;q=0.9,en;q=0.1> on a site that only supports C<en> would currently
 1653: fail to fall back to English.  Use C<I18N::AcceptLanguage> passes only when
 1654: possible.
 1655: 
 1656: =item * B<Logger must be a blessed object>
 1657: 
 1658: The C<logger> parameter is documented as accepting a code ref, array ref, or
 1659: filename, but the current implementation calls C<< $logger->$level() >> and will
 1660: die on non-blessed values.  Wrap alternative logger types in a
 1661: C<Log::Abstraction> instance before passing them to C<new()>.
 1662: 
 1663: =item * B<es-419 sublanguage returns undef>
 1664: 
 1665: Three-part regional codes such as C<es-419> (Latin American Spanish) do not
 1666: resolve to a C<sublanguage()> value because ISO 3166-1 does not define '419'.
 1667: This is a known limitation of the Locale::Object layer.
 1668: 
 1669: =item * B<Whois lookups are slow and unreliable>
 1670: 
 1671: Without C<IP::Country>, C<Geo::IP>, or C<Geo::IPfree> installed, C<country()>
 1672: falls back to Whois queries against live RIPE/ARIN/IANA servers.  These can
 1673: time out under load.  Install at least one local geo-database module and enable
 1674: the CHI cache to avoid this.
 1675: 
 1676: =item * B<Sub::Private not yet enforced>
 1677: 
 1678: The C<_*> private methods are currently accessible from outside the package.
 1679: C<Sub::Private> should be added to enforce encapsulation once white-box tests
 1680: are updated to call only the public API.
 1681: 
 1682: =item * B<EU country code is irresolvable (with one exception)>
 1683: 
 1684: IP addresses that Whois reports as country C<EU> are mapped to C<'Unknown'>
 1685: unless they fall within Baidu's known subnet (RT-86809).  There is no ISO
 1686: 3166-1 country code for the European Union.
